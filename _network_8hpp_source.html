3 // Here's how networking works:
4 //
5 // Vision: Shared or simulated.
6 // Shared vision is the system used in competition. It sends multicast data
7 // to 224.5.20.2 port 10002.
8 // Simulated vision is sent by our simulator to 127.0.0.1 ports 10000 and
9 // 10001.
10 // The data sent to the two simulated vision ports is identical
11 // It's sent to two ports so we can stay on localhost since multicast
12 // doesn't work if there are no routing entries (a common case during
13 // development).
14 //
15 // Radio:
16 // There are two radio channels from the network's point of view.
17 // Currently the hardware only supports one, but this needs to be fixed.
18 // Radio packets stay on localhost.
19 //
20 // Soccer uses ports 12000 and 12001.
21 // Radio uses ports 13000 and 13001.
22 //
23 // RadioTx packets go from soccer to radio.
24 // RadioRx packets go from radio to soccer.
25 //
26 // The network ports are set in Processor's constructor and don't change after
27 // that. They are determined by command-line options (-sim and -r). If no radio
28 // channel is given on the command line, the first available one is picked based
29 // on which soccer-side port can be bound.
